# Ship MDViewer: a cross-platform Tauri 2 desktop app that renders .md files, supports threaded comments anchored to selected text, and exchanges document plus comments via files alone — delivered in three sequential phases (viewer + comments, editing + reattachment, CRDT collaboration + distribution). Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use orchestrate

**Goal:** Ship MDViewer: a cross-platform Tauri 2 desktop app that renders .md files, supports threaded comments anchored to selected text, and exchanges document plus comments via files alone — delivered in three sequential phases (viewer + comments, editing + reattachment, CRDT collaboration + distribution).
**Architecture:** Tauri 2 application with a Rust backend (markdown parsing via pulldown-cmark with data-src-offset annotations, syntect highlighting, anchor model with W3C TextQuoteSelector + TextPositionSelector, comments store, sidecar IO, settings, file watcher) and a TypeScript frontend (vanilla TS view layer rendering the WebView UI, theme variables, IPC adapters). Phase 1 ships a JSON sidecar (schema_version 1); Phase 3 migrates to Automerge CRDT (schema_version 2) and adds conflict-diff plus per-OS bundling.
**Tech Stack:** Rust (tauri 2, pulldown-cmark, syntect, notify, serde, anyhow, tracing, automerge, diff-match-patch-rs/dissimilar — Phase-2 selection), TypeScript + Vite, Vitest (frontend unit), cargo llvm-cov (Rust coverage), Playwright (E2E), Mermaid (lazy-loaded).
