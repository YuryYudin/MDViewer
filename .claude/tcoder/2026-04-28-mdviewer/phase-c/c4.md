# C4: Per-OS bundling pipeline (.dmg / .msi / .AppImage / .deb)

**Avoid:**
- Do NOT add code signing in this task. WHY: success criterion 10 explicitly notes "code signing not required" for the macOS artifact, and adding signing requires committing certs to CI which has no place in v1. Leave a follow-up note in the workflow file.
- Do NOT cross-compile. WHY: each OS WebView (WKWebView, WebView2, WebKitGTK) has runtime dependencies that only resolve correctly when built on its native host. Use one runner per target.
- Do NOT auto-publish on every push. WHY: a typo'd commit on `main` would tag a release. Trigger the workflow on `release` published events (or on tag `v*` push) so packaging is opt-in.
- Do NOT silently change `tauri.conf.json` security settings to enable bundling. WHY: any CSP or capability change is a security-review surface. Confine this task to `bundle.targets`, `bundle.icon`, and the workflow YAML.

## Steps

### Step 1: Configure bundle targets in tauri.conf.json

Modify `src-tauri/tauri.conf.json` `bundle` section:

```json
{
  "bundle": {
    "active": true,
    "targets": ["dmg", "msi", "appimage", "deb"],
    "icon": [
      "icons/32x32.png",
      "icons/128x128.png",
      "icons/128x128@2x.png",
      "icons/icon.icns",
      "icons/icon.ico"
    ],
    "category": "Productivity",
    "shortDescription": "Markdown viewer with collaborative comments",
    "longDescription": "Open markdown files, add threaded comments anchored to selected text, and exchange document plus comments via files alone — no server, no account."
  }
}
```

Place placeholder icon files under `src-tauri/icons/` (Tauri ships a generator: `npm run tauri icon path/to/source.png`).

### Step 2: Add release workflow

Create `.github/workflows/release.yml`:

```yaml
name: Release
on:
  push:
    tags: ['v*']
  workflow_dispatch:

permissions:
  contents: write

jobs:
  bundle:
    strategy:
      fail-fast: false
      matrix:
        include:
          - os: macos-latest
            target: dmg
          - os: windows-latest
            target: msi
          - os: ubuntu-latest
            target: 'appimage,deb'
    runs-on: ${{ matrix.os }}
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '22'
          cache: 'npm'
      - uses: dtolnay/rust-toolchain@stable
      - name: Linux deps
        if: matrix.os == 'ubuntu-latest'
        run: |
          sudo apt-get update
          sudo apt-get install -y libgtk-3-dev libwebkit2gtk-4.1-dev librsvg2-dev libayatana-appindicator3-dev
      - run: npm ci
      - run: npm run tauri -- build --bundles ${{ matrix.target }}
      - uses: actions/upload-artifact@v4
        with:
          name: mdviewer-${{ matrix.os }}
          path: src-tauri/target/release/bundle/**

  # TODO: code signing — macOS notarization (APPLE_*) and Windows EV cert
  #       deferred until after v1 ships per success-criterion-10.
```

### Step 3: Add bundle script to package.json

Modify `/Users/jjb/Work/Projects/MDViewer/.claude/worktrees/mdviewer/package.json` `scripts`:

```json
"build:dmg":      "tauri build --bundles dmg",
"build:msi":      "tauri build --bundles msi",
"build:appimage": "tauri build --bundles appimage",
"build:deb":      "tauri build --bundles deb",
"build:debug":    "tauri build --debug"
```

### Step 4: Verify the host build works

```bash
npm run tauri -- build --debug
```

Expect a debug binary at `src-tauri/target/debug/bundle/...` for the host platform's targets. The full release pipeline only validates on its native runners; locally only the host bundle is produced.

### Step 5: Commit

```bash
git add src-tauri/tauri.conf.json src-tauri/icons .github/workflows/release.yml package.json
git commit -m "C4: per-OS bundling targets + release workflow"
```
