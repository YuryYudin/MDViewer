# C2: Conflict diff (Rust hunks + side-by-side view)

**Avoid:**
- Do NOT compute the diff in TypeScript. WHY: the design's data flow places domain logic in Rust; the conflict module is named in `Rust core`. Putting the diff in JS would split logic across the boundary and force re-implementation if a future CLI subcommand needs the same hunks.
- Do NOT emit character-level hunks. WHY: hand-edit and Accept Left / Accept Right at character granularity is unusable in the wireframe; users review by line. Use a line-anchored diff (the `similar` crate's `TextDiff::from_lines` is ideal) and emit hunks of `(local_range, incoming_range, classification)`.
- Do NOT block conflict detection on the file watcher. WHY: a conflict can arise when the user opens a doc whose sidecar mtime indicates an incoming version diverges from a saved local copy; the conflict module must be invokable on demand, not only on filesystem events.
- Do NOT silently lose unanchored choices. WHY: if a hunk is left at default and the user clicks Finish, the saved bytes must be deterministic — pick "keep local" by default and surface the unresolved hunk count in the wireframe-08 status bar so users notice.

```rust
use serde::{Deserialize, Serialize};
use similar::{ChangeTag, TextDiff};

#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize, ts_rs::TS)]
#[ts(export)]
#[serde(rename_all = "snake_case")]
pub enum HunkKind { Conflicting, Added, Removed }

#[derive(Debug, Clone, Serialize, Deserialize, ts_rs::TS)]
#[ts(export)]
pub struct Hunk {
    pub kind: HunkKind,
    pub local_text: String,
    pub incoming_text: String,
    pub local_range: (usize, usize),     // line numbers (start_inclusive, end_exclusive)
    pub incoming_range: (usize, usize),
}

pub fn diff_md(local: &str, incoming: &str) -> Vec<Hunk> {
    let diff = TextDiff::from_lines(local, incoming);
    let mut hunks = Vec::new();
    let mut local_line = 0usize;
    let mut incoming_line = 0usize;
    let mut buffered: Option<Hunk> = None;

    for change in diff.iter_all_changes() {
        match change.tag() {
            ChangeTag::Equal => {
                if let Some(h) = buffered.take() { hunks.push(h); }
                local_line += 1;
                incoming_line += 1;
            }
            ChangeTag::Delete => {
                let h = buffered.get_or_insert_with(|| Hunk {
                    kind: HunkKind::Removed,
                    local_text: String::new(), incoming_text: String::new(),
                    local_range: (local_line, local_line),
                    incoming_range: (incoming_line, incoming_line),
                });
                h.local_text.push_str(change.value());
                h.local_range.1 = local_line + 1;
                local_line += 1;
                if matches!(h.kind, HunkKind::Added) { h.kind = HunkKind::Conflicting; }
            }
            ChangeTag::Insert => {
                let h = buffered.get_or_insert_with(|| Hunk {
                    kind: HunkKind::Added,
                    local_text: String::new(), incoming_text: String::new(),
                    local_range: (local_line, local_line),
                    incoming_range: (incoming_line, incoming_line),
                });
                h.incoming_text.push_str(change.value());
                h.incoming_range.1 = incoming_line + 1;
                incoming_line += 1;
                if matches!(h.kind, HunkKind::Removed) { h.kind = HunkKind::Conflicting; }
            }
        }
    }
    if let Some(h) = buffered { hunks.push(h); }
    hunks
}
```

### Step 3: Implement Conflict.ts

Create `src/views/Conflict.ts` rendering wireframe 08 via `createElement`. Import the generated `Hunk` type instead of hand-writing it — keeping a single source of truth was the whole point of A2b.

```ts
import type { Hunk, Ipc } from '../ipc';

export async function mountConflict(
  root: HTMLElement,
  ipc: Ipc & { saveDocument(path: string, contents: string): Promise<void>; diffMd(local: string, incoming: string): Promise<Hunk[]> },
  args: { tabId: string; path: string; local: string; incoming: string },
): Promise<void> {
  root.replaceChildren();
  const view = document.createElement('section');
  view.setAttribute('data-view', 'conflict');

  const hunks = await ipc.diffMd(args.local, args.incoming);
  const choices: ('local' | 'incoming' | string)[] = hunks.map(() => 'local');

  hunks.forEach((h, i) => {
    const row = document.createElement('article');
    row.setAttribute('data-hunk-index', String(i));

    const left = document.createElement('pre'); left.className = 'left';
    left.textContent = h.local_text;
    const right = document.createElement('pre'); right.className = 'right';
    right.textContent = h.incoming_text;

    const acceptL = document.createElement('button');
    acceptL.setAttribute('data-action', 'accept-left'); acceptL.textContent = 'Accept Left';
    acceptL.addEventListener('click', () => { choices[i] = 'local'; row.classList.remove('chose-right'); row.classList.add('chose-left'); });

    const acceptR = document.createElement('button');
    acceptR.setAttribute('data-action', 'accept-right'); acceptR.textContent = 'Accept Right';
    acceptR.addEventListener('click', () => { choices[i] = 'incoming'; row.classList.remove('chose-left'); row.classList.add('chose-right'); });

    const handEdit = document.createElement('button');
    handEdit.setAttribute('data-action', 'hand-edit'); handEdit.textContent = 'Hand-edit';
    handEdit.addEventListener('click', () => {
      const ta = document.createElement('textarea');
      ta.value = (choices[i] === 'incoming' ? h.incoming_text : h.local_text);
      ta.addEventListener('input', () => { choices[i] = ta.value; });
      row.appendChild(ta);
    });

    row.append(left, right, acceptL, acceptR, handEdit);
    view.appendChild(row);
  });

  const finish = document.createElement('button');
  finish.setAttribute('data-action', 'finish-merge'); finish.textContent = 'Finish merge';
  finish.addEventListener('click', async () => {
    const merged = mergeBytes(args.local, args.incoming, hunks, choices);
    await ipc.saveDocument(args.path, merged);
    view.dispatchEvent(new CustomEvent('conflict-resolved', { bubbles: true, detail: { path: args.path } }));
  });
  view.appendChild(finish);

  root.appendChild(view);
}

function mergeBytes(local: string, incoming: string, hunks: Hunk[], choices: string[]): string {
  // Walk both sides line-by-line. Hunks have `local_range` and `incoming_range`
  // as (startLineInclusive, endLineExclusive). Outside any hunk, lines are
  // identical in both sides — copy from `local`. Inside each hunk, emit the
  // chosen replacement (`local` text, `incoming` text, or the user's hand-edit
  // if `choices[i]` is neither 'local' nor 'incoming').
  const localLines = local.split('\n');
  const incomingLines = incoming.split('\n');
  const out: string[] = [];
  let localCursor = 0;
  for (let i = 0; i < hunks.length; i++) {
    const h = hunks[i];
    const choice = choices[i];
    // Copy unchanged lines preceding this hunk from local.
    while (localCursor < h.local_range[0]) {
      out.push(localLines[localCursor++]);
    }
    if (choice === 'local') {
      for (let l = h.local_range[0]; l < h.local_range[1]; l++) out.push(localLines[l]);
    } else if (choice === 'incoming') {
      for (let l = h.incoming_range[0]; l < h.incoming_range[1]; l++) out.push(incomingLines[l]);
    } else {
      // Hand-edited string from the textarea — split into lines and emit.
      out.push(...choice.split('\n'));
    }
    // Skip the local hunk lines in the cursor.
    localCursor = h.local_range[1];
  }
  // Tail: copy remaining unchanged local lines.
  while (localCursor < localLines.length) {
    out.push(localLines[localCursor++]);
  }
  return out.join('\n');
}
```

Add a Vitest case asserting two-hunk mixed-choice output: hunk 1 → 'local' keeps the local text; hunk 2 → 'incoming' substitutes the incoming text; surrounding context lines are preserved verbatim. Without this case the merge silently drops mixed-choice scenarios.

### Step 3b: Wire automatic conflict detection on open

Criterion 7 says the conflict view appears when an opened `.md` diverges from the locally saved version — this must happen without the user clicking anything. The routing is split between `Workspace` (detects divergence) and the IPC layer (which has the `AppHandle`):

1. **Modify `src-tauri/src/workspace.rs`** — A8a already defines `OpenOutcome::Document | Conflict { tab_id, path, local, incoming }` and ships an unused `Tab::last_saved_snapshot: Option<String>` field. C2 only updates the body of `open_document` to read the snapshot and return the `Conflict` variant when divergence is detected; it does NOT redefine the enum.

    Inside `open_document`, after `let source = std::fs::read_to_string(&canonical)?;` but before constructing the new tab, check the existing tab's last-saved snapshot when one is being reopened. For closed-but-previously-saved files, persist a per-Workspace `HashMap<PathBuf, String>` (sibling to `tabs`) so the snapshot survives `close_tab`. Pseudocode:

    ```rust
    // A8a's struct gains:
    //   closed_snapshots: HashMap<PathBuf, String>
    // populated from `last_saved_snapshot` on close_tab.

    if let Some(prior) = self.closed_snapshots.get(&canonical) {
        if &source != prior {
            let id = format!("tab-{:x}", std::time::SystemTime::now()
                .duration_since(std::time::UNIX_EPOCH).unwrap_or_default().as_nanos());
            return Ok(OpenOutcome::Conflict {
                tab_id: id,
                path: canonical.clone(),
                local: prior.clone(),
                incoming: source,
            });
        }
    }
    ```

    Concretely, the changes to `Workspace` (in this task's modify list — `src-tauri/src/workspace.rs`) are:

    ```rust
    pub struct Workspace {
        // ... existing fields ...
        // NEW (C2): persists the last-saved snapshot across close_tab so a
        // subsequent open of the same path can detect external divergence.
        closed_snapshots: HashMap<PathBuf, String>,
    }

    impl Workspace {
        pub fn new(data_dir: &Path) -> Result<Self> {
            Ok(Self {
                // ... existing init ...
                closed_snapshots: HashMap::new(),
            })
        }

        pub fn close_tab(&mut self, id: &str) -> Result<()> {
            if let Some(tab) = self.tabs.remove(id) {
                if let Some(snap) = tab.last_saved_snapshot {
                    self.closed_snapshots.insert(tab.path.clone(), snap);
                }
            }
            self.order.retain(|x| x != id);
            if self.active.as_deref() == Some(id) {
                self.active = self.order.last().cloned();
            }
            Ok(())
        }

        // open_document gains the divergence check shown above; the active-tab
        // early-return path also compares disk bytes against last_saved_snapshot
        // and emits Conflict when they differ (otherwise opening an
        // already-open file with a divergent on-disk copy silently activates
        // the stale tab).
    }
    ```

    The `save_document` IPC handler in `src-tauri/src/main.rs` (B3) needs a one-line change to also populate `closed_snapshots`:

2. **Modify `src-tauri/src/main.rs`** — the `open_document` IPC command now consumes `OpenOutcome` and emits the `show-conflict` event from the IPC layer where `AppHandle` is in scope:

    ```rust
    #[tauri::command]
    fn open_document(
        app: tauri::AppHandle,
        state: State<'_, Ws>,
        path: PathBuf,
    ) -> Result<OpenOutcome, String> {
        let outcome = state.lock().map_err(|e| e.to_string())?
            .open_document(&path, OpenOpts::default())
            .map_err(|e| e.to_string())?;
        if let OpenOutcome::Conflict { tab_id, path, local, incoming } = &outcome {
            let _ = app.emit("show-conflict", serde_json::json!({
                "tab_id": tab_id, "path": path, "local": local, "incoming": incoming,
            }));
        }
        Ok(outcome)
    }
    ```

    `Workspace` does not need an `AppHandle`. The IPC handler returns `OpenOutcome` to the frontend so it can render either a Document or a Conflict view based on the discriminant.
