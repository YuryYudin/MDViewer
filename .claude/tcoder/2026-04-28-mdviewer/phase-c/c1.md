# C1: Automerge sidecar (schema_version 2) with migration from v1

**Avoid:**
- Do NOT replace v1 sidecars on disk eagerly during load. WHY: a user's first read after upgrading should not silently rewrite their files — that turns "I just opened a doc" into a destructive action and would confuse anyone collaborating across MDViewer versions. Migrate in memory; rewrite on the next save.
- Do NOT generate fresh thread IDs during migration. WHY: Phase 1 sidecars already share thread IDs with collaborators by file exchange; reissuing IDs would cause every existing thread to look "new" to the counterpart's CRDT and produce duplicates after the first auto-merge.
- Do NOT store the Automerge document as JSON. WHY: the design says "compact Automerge bytes" — Automerge's `save()` produces a binary blob with full operation history. Round-tripping through JSON would force lossy field-by-field reserialization and discard the CRDT history, defeating the merge-order-independence guarantee.
- Do NOT skip the merge-order-independence test. WHY: the success criterion specifically promises "the merged result contains both sets of threads with no data lost"; without applying ops `{A,B}` vs `{B,A}` and asserting equality, a regression in Automerge usage (e.g., wrong actor IDs) would silently lose threads.

```rust
    let mut ba_ids: Vec<_> = ba.list_threads().iter().map(|t| t.id.clone()).collect();
```

### Step 3: Implement Automerge-backed sidecar

Modify `src-tauri/src/sidecar.rs`:

```rust
use automerge::{transaction::Transactable, AutoCommit, ObjType, ReadDoc, ROOT};
use base64::Engine;

#[derive(Serialize, Deserialize)]
struct EnvelopeV2 {
    schema_version: u32,
    automerge: String, // base64
}

pub fn save_sidecar(path: &Path, store: &CommentsStore) -> Result<()> {
    if let Some(p) = path.parent() { std::fs::create_dir_all(p)?; }
    let mut doc = AutoCommit::new();
    let threads_id = doc.put_object(ROOT, "threads", ObjType::List)?;
    for (i, t) in store.list_threads().iter().enumerate() {
        let tid = doc.insert_object(&threads_id, i, ObjType::Map)?;
        doc.put(&tid, "id", t.id.clone())?;
        doc.put(&tid, "resolved", t.resolved)?;
        // ... serialize anchor + comments fields similarly
    }
    let bytes = doc.save();
    let env = EnvelopeV2 {
        schema_version: 2,
        automerge: base64::engine::general_purpose::STANDARD.encode(&bytes),
    };
    std::fs::write(path, serde_json::to_string_pretty(&env)?)?;
    Ok(())
}

pub fn load_sidecar(path: &Path) -> Result<CommentsStore> {
    if !path.exists() { return Ok(CommentsStore::new()); }
    let bytes = std::fs::read(path)?;
    // Try v2 envelope first; fall back to v1 plain JSON.
    if let Ok(env) = serde_json::from_slice::<EnvelopeV2>(&bytes) {
        if env.schema_version == 2 {
            let am_bytes = base64::engine::general_purpose::STANDARD.decode(&env.automerge)?;
            let doc = AutoCommit::load(&am_bytes)?;
            return Ok(automerge_to_store(&doc));
        }
    }
    // v1 path: parse OnDiskV1 (the structure A7 used).
    let v1: OnDiskV1 = serde_json::from_slice(&bytes)?;
    if v1.schema_version != 1 {
        anyhow::bail!("unsupported schema_version {}", v1.schema_version);
    }
    // Construct a CommentsStore preserving every thread/comment/reply ID.
    Ok(CommentsStore::from_threads(v1.threads))
}
```

Modify `src-tauri/src/comments.rs` — expose `merge_stores(local, incoming) -> CommentsStore` that:
1. Reads the Automerge bytes from each store (cache them inside `CommentsStore` after load),
2. `doc.merge(&mut other_doc)` (Automerge's deterministic merge),
3. Walks the merged doc back to a `Vec<Thread>`.
