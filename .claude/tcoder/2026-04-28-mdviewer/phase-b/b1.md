# B1: Diff-match-patch crate selection and fuzzy reattachment

**Avoid:**
- Do NOT lock the diff-match-patch crate before benchmarking. WHY: the design's Key Decisions explicitly defers this choice to Phase 2 because two acceptable Rust ports (`diff-match-patch-rs` and `dissimilar`) exist with different feature/perf trade-offs; locking before measuring contradicts the documented rationale.
- Do NOT pull in `dmp_rs`. WHY: the design's Key Decisions section explicitly rejects it for sparse maintenance / last-publish risk.
- Do NOT change the `Anchor` shape A6 introduced. WHY: Phase 1 sidecars (schema_version 1) on disk reference these field names; renaming or restructuring breaks v1 round-trip and the migration path C1 will rely on.
- Do NOT bypass the `reattachment_confidence` setting. WHY: the Settings → consumer mapping says this exact setting feeds `anchor.rs`; reading it from anywhere else (or hard-coding 75) defeats the user-tunable knob the e2e test in `09-settings-take-effect.spec.ts` exercises.

```rust
use mdviewer_lib::anchor::{resolve_anchor_with_threshold, Anchor, ResolveOutcome};

fn anchor(start: usize, end: usize, exact: &str, prefix: &str, suffix: &str) -> Anchor {
    Anchor { start, end, exact: exact.into(), prefix: prefix.into(), suffix: suffix.into() }

#[test]
fn fuzzy_reattaches_after_minor_insertion() {
    // Original: "selectable phrase one." → user inserts " short" before "phrase".
    let src = "Hello selectable short phrase one. More text.";
    let a = anchor(6, 28, "selectable phrase one.", "Hello ", " More text.");
    let out = resolve_anchor_with_threshold(src, &a, 60);
    match out {
        ResolveOutcome::Resolved { start, end } => {
            assert!(&src[start..end].contains("phrase one"));
        }
        _ => panic!("expected fuzzy resolve, got {out:?}"),
    }
}

#[test]
fn orphans_when_fuzzy_score_below_threshold() {
    let src = "Completely unrelated paragraph with no overlap.";
    let a = anchor(6, 28, "selectable phrase one.", "Hello ", " More text.");
    let out = resolve_anchor_with_threshold(src, &a, 75);
    assert_eq!(out, ResolveOutcome::Orphan);
}

#[test]
fn unicode_boundary_regression() {
    // Anchor crosses an emoji; ensure we never panic on slice-by-byte.
    let src = "Hello 🌍 selectable phrase one. More text.";
    let a = anchor(0, 0, "phrase one", "selectable ", ". More");
    let out = resolve_anchor_with_threshold(src, &a, 75);
    assert!(matches!(out, ResolveOutcome::Resolved { .. }));
}

#[test]
fn property_inserts_deletes_replacements() {
    let original = "alpha bravo charlie delta echo foxtrot";
    let a = anchor(6, 11, "bravo", "alpha ", " charlie");
    // Insertion before the quote.
    let edited = "PRE alpha bravo charlie delta echo foxtrot";
    assert!(matches!(resolve_anchor_with_threshold(edited, &a, 75), ResolveOutcome::Resolved { .. }));
    // Deletion of an unrelated word.
    let edited = "alpha bravo charlie echo foxtrot";
    assert!(matches!(resolve_anchor_with_threshold(edited, &a, 75), ResolveOutcome::Resolved { .. }));
    // Replacement of the quote itself.
    let edited = "alpha XXXXX charlie delta echo foxtrot";
    assert_eq!(resolve_anchor_with_threshold(edited, &a, 90), ResolveOutcome::Orphan);
}
```

### Step 4: Implement fuzzy reattachment

Add `resolve_anchor_with_threshold(source, anchor, confidence_threshold)` to `src-tauri/src/anchor.rs`. Behavior:

```rust
use diff_match_patch_rs::DiffMatchPatch;

pub fn resolve_anchor_with_threshold(source: &str, anchor: &Anchor, threshold_pct: u8) -> ResolveOutcome {
    // Step 1: try exact resolve (the Phase-1 path) first.
    if let ResolveOutcome::Resolved { start, end } = resolve_anchor(source, anchor) {
        return ResolveOutcome::Resolved { start, end };
    }
    // Step 2: fuzzy match using Bitap.
    let mut dmp = DiffMatchPatch::new();
    // match_threshold: 0.0 = exact, 1.0 = anything. Translate confidence_threshold.
    dmp.set_match_threshold(1.0 - (threshold_pct as f32 / 100.0));
    // Clamp the start hint — `anchor.start` is a stale offset from a prior
    // save and may exceed `source.len()` after deletions; some Bitap impls
    // clamp internally and others panic, so we guard here.
    let start_hint = anchor.start.min(source.len());
    let loc = dmp.match_main(source, &anchor.exact, start_hint);
    let Some(start) = loc else { return ResolveOutcome::Orphan; };
    let end = start + anchor.exact.len();
    if end > source.len() || !source.is_char_boundary(start) || !source.is_char_boundary(end) {
        return ResolveOutcome::Orphan;
    }
    // Score: penalize prefix/suffix mismatch.
    let pre_score = matching_suffix_len(&source[..start], &anchor.prefix);
    let suf_score = matching_prefix_len(&source[end..], &anchor.suffix);
    let context_len = anchor.prefix.len() + anchor.suffix.len();
    let context_score_pct = if context_len == 0 { 100 } else { ((pre_score + suf_score) * 100 / context_len) as u8 };
    if context_score_pct < threshold_pct {
        return ResolveOutcome::Orphan;
    }
    ResolveOutcome::Resolved { start, end }
}
```

Add a regression test for the stale-offset case in `src-tauri/tests/anchor.rs`:

```rust
#[test]
fn fuzzy_resolve_handles_stale_start_past_source_end() {
    // Original document had the phrase near offset 200; user then deleted
    // most of the document and the phrase now lives near offset 30. Stale
    // anchor.start of 200 must not panic the resolver.
    let source = "shortened text — the moved phrase still appears here.";
    let anchor = Anchor {
        start: 200, end: 218,
        exact: "the moved phrase".into(),
        prefix: "— ".into(),
        suffix: " still".into(),
    };
    match resolve_anchor_with_threshold(source, &anchor, 75) {
        ResolveOutcome::Resolved { start, .. } => assert!(start < source.len()),
        ResolveOutcome::Orphan => panic!("should have reattached after stale offset"),
    }
}
```

### Step 4b: Wire the threshold from settings into the IPC path

The Phase-1 `resolve_anchor_for_tab` (in `src-tauri/src/workspace.rs`, defined by A8) calls `anchor::resolve_anchor` directly. Widen it to read the user-configured threshold from settings and dispatch through `resolve_anchor_with_threshold`. Replace A8's body with:

```rust
// in src-tauri/src/workspace.rs
pub fn resolve_anchor_for_tab(&self, tab_id: &str, a: &Anchor) -> Result<ResolveOutcome> {
    let tab = self.tabs.get(tab_id)
        .ok_or_else(|| anyhow::anyhow!("no such tab: {tab_id}"))?;
    let threshold = self.settings.get().comments.reattachment_confidence;
    Ok(anchor::resolve_anchor_with_threshold(&tab.source, a, threshold))
}
```

The IPC command in `src-tauri/src/main.rs` is unchanged (still `state.lock().unwrap().resolve_anchor_for_tab(...)`); only the Workspace method body updates. Add a unit test in `src-tauri/tests/workspace.rs` that mutates `settings.comments.reattachment_confidence` to 95 and asserts that an anchor resolvable at 75 is now classified as orphan.
