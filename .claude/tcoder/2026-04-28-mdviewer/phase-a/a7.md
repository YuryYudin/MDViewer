# A7: Comments store and JSON sidecar IO (schema_version 1)

**Avoid:**
- Do NOT bake the sidecar filename into the comments module. WHY: the design's Settings → consumer mapping says the `sidecar_pattern` (default `{name}.md.comments.json`) is owned by `sidecar.rs`. If `comments.rs` builds the path itself, changing the pattern in Settings won't take effect for already-open documents.
- Do NOT use `automerge` here. WHY: Phase rationale says Phase 1 ships plain JSON with `schema_version: 1`; C1 introduces Automerge with a v1→v2 migration round-trip test. Adding automerge now would force a Phase-3 dependency into the Phase-1 binary and the migration test would have nothing to migrate from.
- Do NOT couple resolve_thread to a wall-clock from outside. WHY: tests need deterministic timestamps. Accept `now: SystemTime` as a parameter (or use a simple `Clock` trait with a default `SystemClock` impl) so tests can pin `resolved_at`.
- Do NOT silently coalesce two threads with the same anchor on load. WHY: the design says auto-merge mode `Always = newest mtime wins` for whole-sidecar replacement, but within a single sidecar, distinct thread IDs must remain distinct even if their anchors overlap. Otherwise reply IDs would dangle when merged with an incoming sidecar in C1.

### Step 2: Implement comments.rs

Create `src-tauri/src/comments.rs`:

```rust
use crate::anchor::Anchor;
use serde::{Deserialize, Serialize};
use std::sync::Mutex;
use std::time::SystemTime;

#[derive(Debug, Clone, Serialize, Deserialize, PartialEq, Eq, ts_rs::TS)]
#[ts(export)]
pub struct Comment {
    pub id: String,
    pub author: String,
    pub color: String,
    pub body: String,
    pub created_at: String, // RFC3339
}

#[derive(Debug, Clone, Serialize, Deserialize, PartialEq, Eq, ts_rs::TS)]
#[ts(export)]
pub struct Thread {
    pub id: String,
    pub anchor: Anchor,
    pub comments: Vec<Comment>,
    #[serde(default)]
    pub resolved: bool,
    #[serde(default)]
    pub resolved_at: Option<String>,
    #[serde(default)]
    pub resolved_by: Option<String>,
}

#[derive(Debug, Clone, Serialize, Deserialize, ts_rs::TS)]
#[ts(export)]
pub struct NewThread {
    pub anchor: Anchor,
    pub first_comment: NewComment,
}

#[derive(Debug, Clone, Serialize, Deserialize, ts_rs::TS)]
#[ts(export)]
pub struct NewComment {
    pub author: String,
    pub color: String,
    pub body: String,
}

#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum ChangeEvent { ThreadCreated, ReplyPosted, ThreadResolved, Bulk }

pub struct CommentsStore {
    threads: Vec<Thread>,
    // Same runtime-agnostic fan-out as SettingsStore (A3).
    subs: Mutex<Vec<std::sync::mpsc::Sender<ChangeEvent>>>,
}

impl CommentsStore {
    pub fn new() -> Self {
        Self { threads: Vec::new(), subs: Mutex::new(Vec::new()) }
    }

    pub fn from_threads(threads: Vec<Thread>) -> Self {
        Self { threads, subs: Mutex::new(Vec::new()) }
    }

    pub fn subscribe(&self) -> std::sync::mpsc::Receiver<ChangeEvent> {
        let (tx, rx) = std::sync::mpsc::channel();
        self.subs.lock().unwrap().push(tx);
        rx
    }

    fn emit(&self, ev: ChangeEvent) {
        let mut subs = self.subs.lock().unwrap();
        subs.retain(|tx| tx.send(ev).is_ok());
    }

    pub fn list_threads(&self) -> &[Thread] { &self.threads }
    pub fn get_thread(&self, id: &str) -> Option<&Thread> {
        self.threads.iter().find(|t| t.id == id)
    }

    /// Returns the newly-created Thread by value so the IPC layer can ship
    /// the full structure to the frontend without a second `get_thread` call.
    /// The thread's `id` is also accessible as `thread.id`.
    pub fn create_thread(&mut self, n: NewThread) -> Thread {
        let tid = new_id("t-");
        let cid = new_id("c-");
        let t = Thread {
            id: tid,
            anchor: n.anchor,
            comments: vec![Comment {
                id: cid,
                author: n.first_comment.author,
                color: n.first_comment.color,
                body: n.first_comment.body,
                created_at: now_rfc3339(),
            }],
            resolved: false,
            resolved_at: None,
            resolved_by: None,
        };
        self.threads.push(t.clone());
        self.emit(ChangeEvent::ThreadCreated);
        t
    }

    pub fn post_reply(&mut self, thread_id: &str, n: NewComment) -> Result<(), &'static str> {
        let t = self.threads.iter_mut().find(|t| t.id == thread_id).ok_or("thread not found")?;
        t.comments.push(Comment {
            id: new_id("c-"),
            author: n.author,
            color: n.color,
            body: n.body,
            created_at: now_rfc3339(),
        });
        self.emit(ChangeEvent::ReplyPosted);
        Ok(())
    }

    pub fn resolve_thread(&mut self, thread_id: &str, by: &str) -> Result<(), &'static str> {
        let t = self.threads.iter_mut().find(|t| t.id == thread_id).ok_or("thread not found")?;
        t.resolved = true;
        t.resolved_at = Some(now_rfc3339());
        t.resolved_by = Some(by.into());
        self.emit(ChangeEvent::ThreadResolved);
        Ok(())
    }
}

fn now_rfc3339() -> String {
    use std::time::{Duration, UNIX_EPOCH};
    let n = SystemTime::now().duration_since(UNIX_EPOCH).unwrap_or(Duration::ZERO);
    // Minimal RFC3339-ish stamp; humantime/chrono kept out of the dep list.
    let secs = n.as_secs();
    format!("{}Z", secs)
}

fn new_id(prefix: &str) -> String {
    use std::time::{SystemTime, UNIX_EPOCH};
    let n = SystemTime::now().duration_since(UNIX_EPOCH).unwrap_or_default().as_nanos();
    format!("{prefix}{n:x}")
}
```

### Step 3: Implement sidecar.rs

Create `src-tauri/src/sidecar.rs`:

```rust
use crate::comments::{CommentsStore, Thread};
use anyhow::{Context, Result};
use serde::{Deserialize, Serialize};
use std::path::{Path, PathBuf};

#[derive(Debug, Serialize, Deserialize)]
struct OnDiskV1 {
    schema_version: u32,
    threads: Vec<Thread>,
}

pub fn sidecar_path(md: &Path, pattern: &str) -> PathBuf {
    // Pattern uses `{name}` as a placeholder for the .md filename including its extension.
    let parent = md.parent().unwrap_or(Path::new(""));
    let name = md.file_name().and_then(|s| s.to_str()).unwrap_or("");
    // The default pattern is "{name}.md.comments.json", but the placeholder is the
    // file name *without* its trailing ".md" so the result is stable for any
    // extension casing. To keep it predictable, replace `{name}` with the bare stem.
    let stem = md.file_stem().and_then(|s| s.to_str()).unwrap_or(name);
    let resolved = pattern.replace("{name}", stem);
    parent.join(resolved)
}

pub fn load_sidecar(path: &Path) -> Result<CommentsStore> {
    if !path.exists() {
        return Ok(CommentsStore::new());
    }
    let bytes = std::fs::read_to_string(path).context("read sidecar")?;
    let on_disk: OnDiskV1 = serde_json::from_str(&bytes).context("parse sidecar")?;
    if on_disk.schema_version != 1 {
        // Phase-3 (C1) layers in v2 handling. For Phase-1 we accept v1 only.
        anyhow::bail!("unsupported schema_version {}", on_disk.schema_version);
    }
    Ok(CommentsStore::from_threads(on_disk.threads))
}

pub fn save_sidecar(path: &Path, store: &CommentsStore) -> Result<()> {
    if let Some(parent) = path.parent() { std::fs::create_dir_all(parent)?; }
    let on_disk = OnDiskV1 {
        schema_version: 1,
        threads: store.list_threads().to_vec(),
    };
    let body = serde_json::to_string_pretty(&on_disk)?;
    std::fs::write(path, body).context("write sidecar")?;
    Ok(())
}
```

Auto-merge wiring: when `sidecar.rs` is asked to merge an incoming sidecar (called from B2's watcher in Phase 2 and reused at open time), it consults `settings.comments.auto_merge`:
- `Always` → adopt the file with the newer mtime,
- `Ask` / `Manual` → return a `MergeProposal` value to the caller, which the frontend renders as a prompt.

Stub the policy now as a small enum-returning function so B2 has an attachment point:

```rust
pub enum MergeOutcome { Adopted(CommentsStore), AskUser { local: CommentsStore, incoming: CommentsStore } }
```
