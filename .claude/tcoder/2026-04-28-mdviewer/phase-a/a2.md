# A2: Tauri 2 scaffold (Rust crate, frontend root, build constants)

**Avoid:**
- Do NOT use `cargo tauri init` interactively — the answers it prompts for are baked into the design (app name `MDViewer`, identifier `com.mdviewer.app`, frontend dist `../dist`, dev URL `http://localhost:1420`). WHY: an interactive prompt is unreproducible and makes the implementer guess values the design already pinned.
- Do NOT add Phase 2 / Phase 3 dependencies (`automerge`, `notify`, diff-match-patch crate) to `Cargo.toml` here. WHY: they are scoped to later phases per the design's cumulative dependency table; including them now leaves dead imports and bloats the binary.
- Do NOT register IPC commands yet beyond a single `app_info` smoke command. WHY: A3–A8 add the real commands and need `main.rs` to be small and stable so their `modify` operations are surgical.
- Do NOT hand-roll the version/commit constants in `main.rs`. WHY: `build.rs` is the single source — manual constants drift from the actual git revision.

### Step 3: Write build.rs to emit version/commit constants

Create `src-tauri/build.rs`:

```rust
use std::process::Command;

fn main() {
    tauri_build::build();

    let commit = Command::new("git")
        .args(["rev-parse", "--short", "HEAD"])
        .output()
        .ok()
        .and_then(|o| if o.status.success() { String::from_utf8(o.stdout).ok() } else { None })
        .map(|s| s.trim().to_string())
        .unwrap_or_else(|| "unknown".to_string());
    println!("cargo:rustc-env=MDVIEWER_COMMIT_HASH={}", commit);

    let version = std::env::var("CARGO_PKG_VERSION").unwrap_or_else(|_| "0.0.0".into());
    println!("cargo:rustc-env=MDVIEWER_VERSION={}", version);

    println!("cargo:rerun-if-changed=../.git/HEAD");
    println!("cargo:rerun-if-changed=Cargo.toml");
}
```

### Step 5: Write src/lib.rs (BuildInfo and re-exports)

Create `src-tauri/src/lib.rs`:

```rust
//! MDViewer library crate.
//!
//! This crate is referenced by both `main.rs` (binary entry) and the
//! integration tests under `tests/`. Modules added in later tasks
//! (settings, recents, document, anchor, comments, sidecar, workspace,
//! watcher, conflict) attach here as they are implemented.

#[derive(Debug, Clone, serde::Serialize)]
pub struct BuildInfo {
    pub version: &'static str,
    pub commit_hash: &'static str,
}

pub fn build_info() -> BuildInfo {
    BuildInfo {
        version: env!("MDVIEWER_VERSION"),
        commit_hash: env!("MDVIEWER_COMMIT_HASH"),
    }
}
```

### Step 6: Write src/main.rs (Tauri entry, single smoke command)

Create `src-tauri/src/main.rs`:

```rust
#![cfg_attr(not(debug_assertions), windows_subsystem = "windows")]

use mdviewer_lib::{build_info, BuildInfo};

#[tauri::command]
fn app_info() -> BuildInfo {
    build_info()
}

fn main() {
    tracing_subscriber::fmt()
        .with_env_filter(tracing_subscriber::EnvFilter::from_default_env())
        .init();

    tauri::Builder::default()
        .plugin(tauri_plugin_dialog::init())
        .plugin(tauri_plugin_fs::init())
        .invoke_handler(tauri::generate_handler![app_info])
        .run(tauri::generate_context!())
        .expect("error while running tauri application");
}
```

```ts
import { invoke } from '@tauri-apps/api/core';

interface BuildInfo {
  version: string;
  commit_hash: string;
}

async function main() {
  const info = await invoke<BuildInfo>('app_info');
  const root = document.getElementById('app')!;
  root.textContent = `MDViewer ${info.version} (${info.commit_hash})`;
}

main().catch((err) => {
  document.body.append(`Bootstrap error: ${err}`);
});
```
